Comienzo

Por qué YAML?

  • Porque no se pueden matar moscas a cañonazos.
  • Porque se usa cada vez más.
  • Porque la cualidad principal del usuario de Perl es la pereza.

En el principio

  • Existía Data::Denter.
    • *** Data::Denter #1 *** %Menda::Lerenda Bolsillo => @ Llaves Gorra Zapatos => Gorila

Pero qué diablos es, joé

  • YAML Ain't a Markup Language
  • Un lenguaje de etiquetas sin etiquetas
  • Un lenguaje de serialización simple y ligero.
  • Un estándar público.
  • Muy popular en lenguajes como el Ruby
    • nombre: YAML sirvepara: cosas esmuy: guay

¿Tanto follón pa eso?

  • Pero además
  • Legible.
  • Editable.
  • Multilingüe.
  • A salvo de mancillas (taint-safe).
  • Puede serializar todas las estructuras normales.
  • Extensible a partir de un sistema de tipos.

¿Para qué sirve?

  • Ficheros de configuración
  • Metadatos en CPAN (META.yml).
  • Lista negra de spam en Movable Type.
  • Un CMS lo usa. No recuerdo cuál.

¿Cómo se usa?

  • Parser Syck.
  • Pero YAML::Parser::Syck no es demasiado útil.
  • PyYAML.
  • Módulos YAML::.
  • xyx: XML to YAML (y patrás).
  • ysh: YAML to Perl (y patrás).

Las caenas

  • Simples
    • "Cadena"
  • Multilinea.
    • '"Esto es una cadena con muchas líneas y otra más"'

Acuérdate de esto

  • Referencias
    • - &telocuento Te lo digo - Pepillo - Juanillo - *telocuento
  • Que daría algo así
    • $VAR1 = [ 'Te lo digo', 'Pepillo', 'Juanillo', 'Te lo digo' ];

El marchamo

  • El sistema de proceso de YAML divide los documentos en nodos.
  • Cada nodo tiene un tipo, explícito o implícito.
  • Cadenas
    • !!str 35 pero no es un número
  • Pero puedes meter etiquetas propias.
    • !medallilla del jamón
  • Que crearía un nodo de tipo medallilla.
  • Se usa también para crear nodos específicos de lenguaje
    • !ruby/range 10..20

Manantial de la abundancia

  • Separa documentos.
    • --- -Aqui empieza un documento --- -Aqui empieza el siguiente
  • Incluso se puede terminar un documento
    • --- -Aqui empieza un documento -y termina ... --- -Aqui empieza el siguiente ...

Ahora el camello

  • Varios módulos desde Perl
  • Por supuesto, YAML.
    • use YAML qw(LoadFile); my $yaml = LoadFile('peasoficheroescritoen.yaml');

Pero todavía es más fácil.

  • Uso de Config::YAML.
    • my $cartilla = Config::YAML->new( config => "~/.foorc", mi => "mamá", me => "mima", yo => [ 'amo', 'a', 'mi','mamá' ], );
  • Que daría
    • --- mi: mamá me: mima yo: - amo - a - mi - mamá

Más formas de escribir

  • IO::YAML
    • my $path = shift || 'my.yaml'; $io = IO::YAML->new; $io->open($path, '>') or die $!; my $taca = 'Zapatilla'; my $tiqui = \$taca; bless $tiqui, 'Pantuflo'; my $zipi= { 'esto' => 'es','una' => [ 'estructura',[ { 'algo' => 'complicada'}], 'pero', $tiqui] }; print $io $zipi;
  • Que produciría
    • --- esto: es una: - estructura - - algo: complicada - pero - !perl/$Pantuflo Zapatilla

La joya de la corona

  • Ruby y YAML, una pareja para el mal
    • require 'yaml' puts( [ 'amigueete', 'coleguilla', [ 'y', 'el', 'resto'] ].to_yaml )
  • Que saca lo esperado
    • --- - amigueete - coleguilla - - y - el - resto

No se vayan todavía, aún hay más

  • OkayRPC es un protocolo de llamada remota a procedimientos, similar a XML-RPC.
    • --- !okay/rpc/begin --- !okay/rpc/method store.chargeCreditCard: - name: Jon Adams card: 5011501150115011 expires: 03/2002 price: $13.00 --- !okay/rpc/end
  • Incluso YPath similar a XPath, es decir, un modo de seleccionar nodos de un documento YAML.
  • Sólo está en la fase de Brainstorming.
